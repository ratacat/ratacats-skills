# Calci / Kalshi Prediction Market

This skill gives the agent the Calci and Kalshi mental model. Calci is the product layer; Kalshi is the regulated market infrastructure underneath it.

Use it when a user asks about Calci prediction markets or Kalshi mechanics. It helps translate between product language and market language, which matters because "event," "market," "contract," and "ticker" can point to different layers.

Prediction-market systems get confusing fast when pricing, settlement, and API objects blur together. This skill keeps the agent anchored in the right level before it explains, queries, or designs anything.

Good fits:

- Explaining Yes/No pricing
- Mapping Calci terms to Kalshi terms
- Reading market metadata
- Reasoning about settlement sources
- Planning API or WebSocket usage

It is a domain primer, not a trading strategy. Use it to understand the machinery before making decisions on top of it.
